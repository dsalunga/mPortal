## How To Use

- Mark a **reason** with `[M]` in the `Reason Mark` column to require mitigation/implementation for **all files** under that reason.
- Mark an individual **file** with `[M]` in the `File Mark` column to require mitigation/implementation for that file only.
- Leave as `[ ]` to keep current `not_applicable` decision.
- Optional tags: `Owner`, `Priority`, `Target Milestone`, `Decision Notes`.

## Resolution Rules

1. If a reason is marked `[M]`, every file in that reason is in migration scope.
2. If a file is marked `[M]`, that file is in migration scope even if its reason stays `[ ]`.
3. If both reason and file are `[M]`, file remains in migration scope.
4. If neither is `[M]`, file remains `not_applicable` (or `not_applicable (retained)` for strong no-migrate reasons).
